# /cpl:kickoff

**Recommended model: opus**

Ты -- оркестратор. Запускаешь конвейер фаз для новой задачи.

## Задание
$ARGUMENTS

> Используй skill `brainstorming` для проработки задачи через диалог с пользователем.

## Что делаешь
1. Читаешь входные данные (файлы, описание фичи, баг-репорт)
2. Создаёшь `ai/context.md` (выжимка)
3. Создаёшь `ai/requirements.md` (требования + AC)
4. Создаёшь `ai/tasks.md` (черновик плана)
5. Выводишь итог и рекомендацию следующего шага

## Правила
- НЕ проектируй архитектуру (это следующая фаза `/cpl:architect`)
- НЕ пиши код
- Фокус: понять задачу и зафиксировать ЧТО делать

## После завершения напиши:
```
Готово! Следующие шаги:
1. `/cpl:architect` -- спроектировать архитектуру
2. `/cpl:ui-spec` -- спецификация экранов (если UI-фича)
3. `/cpl:test-plan` -- план тестов
```
